Simple.css

Simple.css is my attempt at the least amount of CSS (in addition to the browser default styles) needed to maximize the readability (clarity and economy of display) of English language web documents on both desktop and mobile devices.

Without further digression, here is simple.css: 

html {padding: 10px 10px 200px; font-family: droid sans, sans-serif; }
body {margin: 0 auto; max-width: 580px; }
div.article > *:first-child {margin-top: 0; }

The page you are reading uses only this CSS, plus some styling on the <pre> elements for the code examples.

There are two HTML requirements to use simple.css. First is the viewport meta tag in the head section (for mobile display.) 

<meta name="viewport" content="initial-scale=1.0, user-scalable=no">

Also, the body content should be wrapped in a div with class value equal to "article." 

<div class="article"></div>

When HTML5 is more widely supported I will change this to: 

<article></article>

Reasoning

The thoughts behind this CSS are based on actual research, my first-hand experience, and some as yet unproven personal preferences. I am certain there are mistakes to be fixed and improvements to be added, but I am happy labelling the above CSS "version 1.0."

Here is my reasoning for each style rule.

Maximum Width

body {max-width: 580px; }

Setting a maximum width may be the most important improvement to readability over default browser styles alone. Choose a random book from your shelf and open to any page. Count the words of a single line of paragraph text. Your measurement should fall in between 10 and 15 (or about 72 characters per line). This was not chosen randomly — the longer a line of text becomes, the easier it is to lose vertical eye position while reading a paragraph.

When the Web was being invented, most screens were much smaller and lower resolution than today. Web documents that looked fine full-screen then, when displayed maximized on a 24-inch monitor are now unreadable, rendering paragraphs as one long line. A body width of 580 pixels is about 72 characters wide in my browser's default paragraph sans-serif font.

For mobile devices the problem is the opposite — increasing readability on a tiny screen. I included the meta viewport element mentioned above to ensure the document makes use of the entire screen width on mobile devices. Using the max-width CSS property instead of width means that smaller devices will ignore its value and stretch across the entire screen.

Font Face

html {font-family: droid sans, sans-serif; }

Traditional serif (extra little lines on the ends of letters) fonts were developed for printed text. Unfortunately, low-resolution monitors cannot render the fine detail needed to display the serifs. Even though screen resolution is high enough today to handle serif font faces, sans-serif fonts are still considered superior for screen readability, if only to support low-resolution devices.

Using the generic font-family sans-serif instead of a specific font face in the CSS declaration, we allow the target device to choose the preferred font (typically Arial for Microsoft products and Helvetica for Apple.) Droid Sans is a very nice sans-serif screen font developed for the Android operating system. Unfortunately it is not set as the default sans-serif font in their mobile browser so I added it specifically to the CSS declaration.

Floating Center

body {margin: 0 auto; }

When a short-width document is displayed maximized in a very large monitor, by default it is flush against the left side. By centering the document content, we are positioning it directly in front of where the user's eyes will probably be. In my opinion, dividing the extra space between both sides of the content also feels more balanced compared to one huge blank area on the right.

Minimal Margins

html {padding: 10px 10px 200px; }

For screen resolutions smaller than 580 pixels, we need to add a little space around the content so it is not smashed up against the top and sides of the browser window. It is also nice to add some extra white space at the end so the last of the content is not stuck to the bottom of the browser. This gives the user the freedom to scroll the final bits to a more natural reading position.

Remove Top Margin from First Element

div.article > *:first-child {margin-top: 0; }

At the beginning of this article, I mentioned that all HTML body content was wrapped in a div element with class "article." There are reasons for this, but the most practical is that it is required to enable "reader mode" support in mobile Safari.

This, in addition to the CSS rule, will make your content begin immediately after the 10px html top padding, instead of after a useless gaping blank spot caused by the browser's default margins.

Paragraph Line Height

div.article p {line-height: 1.5em; }

Browser's default line spacing is a little cramped. Some typographic standards recommend 1.5 times the line height, which I agree looks better so I'll change that.

Hopefully a Good Start

I will be testing this CSS (by using it in my HTML documents and viewing on every device I can find) starting now, and improving it as needed. Keeping with the spirit of this project, I hope there will not be many additions — I would like to keep the CSS as simple as the display of the document content. If you have thoughts, please report an issue or if you like, fork the repository.
